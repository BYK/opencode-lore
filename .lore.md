## Long-term Knowledge

### Gotcha

<!-- lore:019e177d-3806-7c52-84ae-f83519d1413f -->
* **Bun worker.terminate() crashes with fastembed NAPI modules**: Calling \`worker.terminate()\` on a \`node:worker\_threads\` Worker that has loaded fastembed (ONNX Runtime NAPI bindings) causes a fatal NAPI crash in Bun during native binding cleanup. The embedding functionality itself works correctly. Fix: send a shutdown message to the worker and have it call \`process.exit(0)\` internally. Additionally, the shutdown method must return a Promise that resolves only after the worker's \`exit\` event fires — if the process ends before the worker fully exits, Bun still crashes during cleanup.

<!-- lore:019e177d-37f1-77b7-9c53-46cda2e9f403 -->
* **Bun worker.terminate() crashes with NAPI modules (e.g. fastembed/ONNX Runtime)**: Calling \`worker.terminate()\` on a Bun worker that has loaded NAPI native modules (fastembed, ONNX Runtime) causes a fatal NAPI crash during cleanup. Fix: send a shutdown message to the worker and call \`process.exit(0)\` from inside the worker instead. Additionally, the caller must await the worker's \`exit\` event before the process ends — otherwise Bun's test runner forcefully tears down mid-cleanup workers, also triggering the crash. Pattern: \`LocalProvider.shutdown()\` returns a Promise that resolves on the worker's \`exit\` event; tests must \`await\` it in \`afterEach\`.

* **Bun worker.terminate() crashes with NAPI modules (fastembed/ONNX Runtime)**: Calling \`worker.terminate()\` on a Bun worker that has loaded fastembed's native ONNX Runtime NAPI bindings causes a fatal NAPI crash during cleanup. Workaround: send a shutdown message to the worker and call \`process.exit(0)\` from inside the worker. The host must then wait for the worker's \`exit\` event before the process ends — otherwise Bun's process exit forcefully tears down mid-cleanup NAPI state. If \`shutdown()\` is async and awaited in test \`afterEach\`, the crash is avoided. Synchronous \`resetProvider()\` calls in tests that never spawn a real worker (e.g. those using \`\_markFastembedUnavailable()\`) are safe since the returned promise resolves immediately.

* **Bun: worker.terminate() crashes with fastembed NAPI/ONNX bindings**: Calling \`worker.terminate()\` on a Bun Worker that has loaded fastembed (native ONNX Runtime NAPI bindings) causes a fatal NAPI crash during teardown. The ONNX Runtime native bindings don't clean up safely when Bun forcefully terminates the worker. Fix: send a shutdown message to the worker and have it call \`process.exit(0)\` itself, then wait for the worker's \`exit\` event before continuing. If the worker hasn't fully exited before the main process ends (e.g. in test runner teardown), Bun's process exit triggers the same crash.

### Pattern

<!-- lore:019e177d-3810-7722-b1ab-0fd78bd3f5d0 -->
* **Graceful worker shutdown: await exit event before process end**: When using \`node:worker\_threads\` Workers that load NAPI modules (especially native ML/ONNX bindings), \`shutdown()\` must be async and wait for the worker's \`exit\` event — not just send a shutdown message. Pattern: send a message (e.g. \`{ type: 'shutdown' }\`), worker calls \`process.exit(0)\`, host awaits \`new Promise(resolve => worker.once('exit', resolve))\`. In tests, \`afterEach\` must \`await\` the shutdown to ensure the worker fully exits before test runner teardown.

<!-- lore:019e177d-3800-7f65-8dbe-54444c9c4a49 -->
* **Worker shutdown pattern: message-based exit + await exit event**: For workers loading NAPI modules in Bun, never use \`worker.terminate()\`. Instead: (1) send a \`{ type: 'shutdown' }\` message, (2) worker calls \`process.exit(0)\` on receipt, (3) \`LocalProvider.shutdown()\` returns a \`Promise\<void>\` that resolves on the worker's \`exit\` event. Test teardown must \`await provider.shutdown()\` to prevent Bun from crashing when the test process exits before worker cleanup completes.

<!-- lore:019e177d-37f6-7d94-b0e6-35961890587a -->
* **Worker thread shutdown must be awaited before process exit when NAPI modules are loaded**: When a worker thread loads NAPI native bindings, the main thread must wait for the worker's \`exit\` event before process termination. Synchronous or fire-and-forget shutdown causes a segfault in Bun's cleanup path. Make \`shutdown()\` async, send a message to trigger \`process.exit(0)\` in the worker, then resolve on the worker's \`exit\` event. In tests, \`afterEach\` must \`await shutdown()\`.
