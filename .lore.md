### Architecture

* **Cache sacred principle: invalidate only when content changes justify cost**: Core principle for cache management: LTM cache is sacred — treat every invalidation as expensive. Cache busts require Anthropic to write new prefix tokens (cost + latency). Never invalidate preemptively or speculatively. Invalidate only when actual content changes (curation, consolidation, dead-ref cleanup) justify the bust cost. Session-specific invalidations: use \`ltmSessionCache.delete(sessionID)\`. Global invalidations: reserve for project-wide changes. Monitor analytics (hit rate, divergence points) to distinguish legitimate busts (content changed) from unnecessary ones (implementation flaw).

### Gotcha

<!-- lore:019e0856-3918-7f71-9e7a-bb6308ea588f -->
* **Gateway drops signature field from Claude thinking blocks causing API 400 errors**: Claude's thinking blocks require a \`signature\` field, but gateway's \`LoreReasoningPart\` type omits it. Round-trip through gateway drops signature: Claude response → \`parseContent()\` → \`LoreReasoningPart\` (no signature) → \`serializeContent()\` → malformed request → 400 error. \*\*Fix\*\*: Add \`signature?: string\` to \`LoreReasoningPart\` interface in shared/types, update \`parseContent()\` to preserve signature from thinking blocks, update \`serializeContent()\` to include signature when present. Critical for Claude Code compatibility since it uses thinking mode by default.

<!-- lore:019e0855-c3f7-789c-9613-e8a73ae29d57 -->
* **Gateway drops signature field from thinking blocks during message reconstruction**: Gateway's \`temporal-adapter.ts\` converts assistant messages with thinking blocks to \`LoreReasoningPart\` objects that only have \`content\` field—the \`signature\` field from original thinking blocks is lost. When gateway reconstructs messages for upstream API (e.g., in \`buildRecallFollowUp\`), it creates new thinking blocks from \`LoreReasoningPart.content\` but never restores the original \`signature\`. This causes "Field required" errors when Anthropic expects thinking blocks to have signatures. \*\*Fix\*\*: Either preserve signature in \`LoreReasoningPart\` type or ensure reconstructed messages don't include thinking blocks that require signatures.
