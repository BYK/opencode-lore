## Long-term Knowledge

### Architecture

<!-- lore:019e0d91-d610-7727-9a92-20ae7e240510 -->
* **Conversation TTL drives cache cost and write multipliers**: Cache write cost multipliers keyed by conversationTTL from config: shorter TTLs (5m) cost less (1.25x) than longer TTLs (1h, 2x). TTL set in \`AnthropicCacheOptions.conversationTTL\` and used in cache\_control markers during request serialization. Enables fine-grained cost control: shorter conversations use cheaper cache, longer sessions tolerate higher write costs. Config value at \`cache.conversationTTL\` propagates through ModelSpec cacheWriteCost calculations.

### Decision

<!-- lore:019e0d86-d7b4-74e2-b078-2fa3d826ec0e -->
* **Model cost baseline for planning — Opus 4.6 at $175.77/day**: Model cost baseline for planning: Daily costs — Opus 4.6: $175.77, Sonnet 4: $6.37, Haiku 4.5: $7.01. Target $55-80/day (69-75% reduction). Cost-aware context caps calibrated against Opus baseline. Cache write multipliers (1.25x for 5m, 2x for 1h) and batch worker costs ($11/day) are primary cost levers. ModelSpec.cacheWriteCost embeds per-model pricing for dynamic decisions.

### Gotcha

<!-- lore:019e0d92-4bbd-7968-bca7-69e788cf0781 -->
* **Rebuild core dist after changing cache config field access**: Changes to AnthropicCacheOptions or cache config field usage in pipeline.ts require rebuilding the @anthropic-ai/sdk dist. The compiled dist doesn't reflect recent changes to the cache field structure. Run build step on core before type-checking pipeline changes that access cache config fields.
